## Objetivo del taller
El propósito del trabajo fue implementar buenas prácticas de organización, documentación y trazabilidad en un entorno de desarrollo colaborativo. Para ello se integraron herramientas como Quire para la planificación, Git y GitHub para el control de versiones, y Mermaid y Draw.io para la representación del flujo de trabajo del proyecto.

## Estructura del proyecto
En la carpeta data se encuentran los datos utilizados en el proyecto, divididos en dos subcarpetas: en raw se almacenan los datos originales sin modificaciones, mientras que en processed se guardan los datos ya transformados y listos para el análisis.

- En la carpeta results se almacenan los productos generados durante el desarrollo del proyecto. Dentro de esta carpeta, figures contiene los gráficos e imágenes producidas, y tables incluye las tablas derivadas del análisis.

- La carpeta scripts contiene los archivos de código utilizados para ejecutar el flujo de trabajo, incluyendo los procesos de generación de datos, análisis y automatización del proyecto.

- La carpeta src contiene funciones y módulos auxiliares desarrollados para organizar el código de manera modular y facilitar su reutilización.

- En la raíz del repositorio se encuentran los archivos de los diagramas de flujo, el archivo .gitignore, el archivo README y los elementos necesarios para documentar el proyecto.

## Gestión del proyecto con Quire
La planificación del taller se realizó utilizando la herramienta Quire, donde se creó un proyecto específico para organizar todas las tareas necesarias. En esta plataforma se estructuraron las actividades, se utilizaron etiquetas cuando fue pertinente y se realizó un seguimiento continuo del progreso mediante la actualización del estado de las tareas, garantizando una planificación ordenada y documentada.

## Control de versiones con GitHub
El desarrollo del taller se llevó a cabo aplicando principios de control de versiones. Se trabajó con múltiples ramas para separar las etapas de análisis y modelado, se realizaron fusiones controladas hacia la rama principal y se practicó la resolución de conflictos de manera intencional como parte del ejercicio. También se aplicaron buenas prácticas como la recuperación de archivos eliminados y la modificación de commits mediante la opción amend, asegurando un historial limpio y coherente del proyecto.

## Diagramas de flujo con Mermaid
Se construyó un diagrama de flujo del proyecto utilizando Mermaid JS, representando de manera estructurada el flujo de trabajo desarrollado. El código que generó el diagrama se encuentra en el archivo correspondiente dentro del repositorio, y la representación gráfica fue incluida en el documento final del taller.

## Diagramas de flujo con Draw.io
Posteriormente, se elaboró una versión mejorada del mismo flujo de trabajo utilizando la herramienta Draw.io. En esta versión se incorporó una característica adicional inspirada en buenas prácticas de investigación reproducible, fortaleciendo la estructura del proceso. El archivo editable fue incluido en el repositorio junto con su respectiva imagen exportada en el documento final.

## Herramientas utilizadas
Durante el desarrollo del proyecto se utilizaron Git, GitHub, GitHub Desktop, Visual Studio Code, PowerShell, Quire, Mermaid JS y Draw.io, integrando herramientas de gestión, programación y documentación en un flujo de trabajo coherente y reproducible.
